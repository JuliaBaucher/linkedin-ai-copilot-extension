# LinkedIn AI Copilot - Phase 1

LinkedIn success depends on the first 60–90 minutes after publishing. Posts that receive 5–10 thoughtful comments early are far more likely to reach a wider audience—but busy professionals rarely have time to read posts, research linked articles, and write meaningful comments consistently.

LinkedIn AI Copilot solves this problem. It is a unique AI Chrome extension that works on desktop and mobile (via remote desktop), reads LinkedIn posts and linked external articles, and generates personalized, high-quality comments in seconds. It helps busy professionals save time, build stronger communities, increase engagement, and maximize the visibility of their own posts.

## Phase 1 Scope

Built:

- LinkedIn feed posts, individual post pages, and already-open LinkedIn articles.
- Comment box detection with a small Sparkle button near detected comment boxes.
- Floating copilot popup near the comment box.
- LinkedIn context extraction: post text, author when available, hashtags, first external URL.
- Best-effort article reading:
  - Already-open LinkedIn article DOM first.
  - Else the first external URL in a LinkedIn post.
  - Public HTML pages only.
  - 10 second timeout.
  - 8,000 character maximum extracted article text.
- AI generation using the OpenAI Responses API with structured JSON.
- Three comment options: technical insight, business/executive insight, and discussion question.
- Copy, insert into comment box, and regenerate actions.
- Options page for API key, model, tone, language, technical level, and emoji preference.
- Local settings using Chrome Storage API.

Not built in Phase 1:

- Style learning.
- Daily feed assistant.
- Web search or browsing beyond the first external URL.
- Multi-provider AI.
- Post generator.
- Paywalled articles, PDFs, videos, login-only pages, or deep crawling.

## MVP Assumptions

- LinkedIn DOM selectors are inherently unstable. Selector lists are isolated in `src/content/linkedinDom.ts` and include comments where future updates are likely.
- The extension asks for broad `http://*/*` and `https://*/*` host permissions only so the MV3 service worker can attempt first-external-URL article fetches. The content script still runs only on LinkedIn.
- Article extraction is intentionally best effort. If CORS, login walls, paywalls, timeout, unsupported content, or empty content prevent reading, the UI shows `Article could not be read automatically.` and continues with LinkedIn post text.
- The default model is `gpt-4.1-mini`. You can change it from the Options page.
- No OpenAI API key is hardcoded. The key is stored locally in `chrome.storage.local`.

## Load Unpacked Extension in Chrome

1. Open Chrome.
2. Go to `chrome://extensions`.
3. Enable `Developer mode`.
4. Click `Load unpacked`.
5. Select the `dist` folder generated by `npm run build`.
6. Open the extension Options page and add your OpenAI API key.
7. Visit LinkedIn and open a feed post, post page, or article page with comments.

## Testing Guide

Manual test flow:

1. Run `npm run build`.
2. Load `dist` as an unpacked extension in Chrome.
3. Open the extension Options page.
4. Save your OpenAI API key and preferences.
5. Open `https://www.linkedin.com/feed/`.
6. Find a post with a visible comment field.
7. Confirm a Sparkle button appears near the comment field.
8. Click the Sparkle button.
9. Confirm the popup generates a post summary, explanation, and three comment options.
10. Click `Copy` and confirm clipboard content.
11. Click `Insert` and confirm the comment text appears in the comment box.
12. Confirm LinkedIn does not publish anything automatically.
13. Open a LinkedIn article page and repeat.
14. Try a post with an external article URL and confirm article extraction either enriches the result or gracefully falls back.

Failure tests:

- Remove the API key and confirm the popup shows a friendly missing-key error.
- Use a blocked/paywalled URL and confirm the extension still generates from the LinkedIn post text.
- Change the model to an invalid value and confirm API errors are shown without breaking the LinkedIn page.

## Security and Privacy Notes

- The extension sends extracted LinkedIn post/article context to OpenAI only when the user clicks the Sparkle button and requests generation.
- The extension does not use a backend or external database.
- Settings remain in local Chrome extension storage.
- Insert only places generated text in the comment editor. The user must manually review and click Post.

# Using the Extension from a Mobile Phone

Although Chrome extensions do not run directly on mobile Chrome, you can use the extension by remotely controlling your desktop computer with Chrome Remote Desktop.

## One-time setup on your PC

1. Install Chrome Remote Desktop.

   https://remotedesktop.google.com/access

2. Click **Set up remote access**.

3. Install the Chrome Remote Desktop Host if prompted.

4. Give your computer a name.

   Example:

   ```
   julia
   ```

5. Create a secure PIN.

   > Do **not** store your PIN in this repository or share it publicly.

6. Keep your computer:

   - Powered on
   - Connected to the Internet
   - Awake (disable automatic sleep while using remote access)

---

## Install the mobile app

Install **Chrome Remote Desktop** on your phone.

- Android: Google Play Store
- iPhone: Apple App Store

Log in using the **same Google account** used on your desktop.

---

## Connect from your phone

1. Open Chrome Remote Desktop.
2. Select your computer.

   Example:

   ```
   julia
   ```

3. Enter your PIN.
4. Your Windows desktop will appear on your phone.

---

## Using the LinkedIn AI Copilot

Inside the remote desktop session:

1. Open **Chrome** running on your PC.
2. Navigate to:

   ```
   https://www.linkedin.com
   ```

3. Make sure the LinkedIn AI Copilot extension is enabled.
4. Click inside a LinkedIn comment box.
5. Tap the ✨ Sparkle button.
6. Generate AI comments.
7. Choose **Insert**.
8. Review the generated draft.
9. Manually click **Post** on LinkedIn.

The extension never publishes comments automatically.

## Useful Remote Desktop Tips

### Use "Send text"

At the bottom of the Chrome Remote Desktop mobile app you'll find a **Send text** field.

This lets you type using your phone's keyboard and send the text directly to your desktop computer.

It's extremely useful for:

- URLs
- Search queries
- Long prompts
- API keys
- Passwords
- Editing LinkedIn comments

Type

```
https://www.linkedin.com
```
